# CS-360 - Final Project - Weight Tracker App

**Briefly summarize the requirements and goals of the app you developed. What user needs was this app designed to address?**

The goal of my Weight Tracker app was to give users a simple way to record and manage their daily weight. The application allows users to create an account, log in, add weight entries, edit existing entries, delete entries, and enable SMS notifications. I designed the app for people who want an easy way to monitor their progress without using a complicated fitness application. My goal was to create something that was simple, organized, and easy to navigate.

**What screens and features were necessary to support user needs and produce a user-centered UI for the app? How did your UI designs keep users in mind? Why were your designs successful?**

The app includes a login screen, a weight tracking screen, and an SMS notification screen. The login screen allows users to create an account and securely sign in. The weight tracking screen lets users add, update, view, and delete weight entries using a simple layout. The SMS notification screen allows users to choose whether they want to enable text message notifications.

When designing the interface, I tried to keep everything straightforward so users could quickly find the features they needed. I avoided adding unnecessary buttons or information that might make the app confusing. I believe the design was successful because each screen has a clear purpose and users can easily move through the application.

**How did you approach the process of coding your app? What techniques or strategies did you use? How could those techniques or strategies be applied in the future?**

I approached the project by focusing on one feature at a time instead of trying to build the entire application at once. I started with the login system, then created the SQLite database, followed by the weight tracking features and finally the SMS notification functionality. Breaking the project into smaller pieces helped me stay organized and made it easier to solve problems as they came up.

In future projects, I will continue using this approach because it helped me avoid becoming overwhelmed. Completing one feature before moving to the next also made debugging much easier.

**How did you test to ensure your code was functional? Why is this process important, and what did it reveal?**

I tested the application frequently using the Android Emulator in Android Studio. After adding each feature, I made sure it worked before continuing with the next part of the project. I tested creating accounts, logging in, adding weight entries, editing entries, deleting entries, and enabling SMS permissions. Testing each feature helped me find mistakes early and prevented larger problems later in the project. It also gave me confidence that each part of the application was working as expected before submission.

**Consider the full app design and development process from initial planning to finalization. Where did you have to innovate to overcome a challenge?**

One of the biggest challenges was getting all of the different parts of the application to work together. I had to connect the login system, SQLite database, CRUD operations, and SMS permission handling into one working application. There were several times when I had to troubleshoot layout issues and fix problems with navigation between screens. Working through those issues taught me the importance of being patient and solving one problem at a time instead of trying to fix everything at once.

**In what specific component of your mobile app were you particularly successful in demonstrating your knowledge, skills, and experience?**

I believe the strongest part of my project was the database functionality. I successfully implemented SQLite to store user accounts and weight entries while allowing users to create, view, update, and delete records. I was also proud of implementing the login system and SMS permission feature because they helped make the application feel more complete. Completing the project from the initial design through a fully functional application showed how much I have learned about mobile app development throughout this course.
